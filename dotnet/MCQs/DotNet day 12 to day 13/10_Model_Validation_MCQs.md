## EASY MCQs

### Q1. Which namespace contains attributes like `[Required]` and `[StringLength]`?
A. `System.Web`
B. `System.ComponentModel.DataAnnotations`
C. `System.Validation`
D. `Microsoft.Validation`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
This is the standard namespace for validation attributes shared across EF Core and ASP.NET Core.
</details>

### Q2. How do you check in the Controller if validation passed?
A. `if (true)`
B. `if (ModelState.IsValid)`
C. `if (Request.IsValid)`
D. `if (Validation.Check())`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
`ModelState.IsValid` returns true only if there are no errors in the `ModelState` dictionary.
</details>

### Q3. Which Tag Helper displays the error message for a specific field?
A. `asp-validation-summary`
B. `asp-validation-for="PropertyName"`
C. `asp-error`
D. `asp-alert`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Renders a `<span>` with the error message associated with the specified model property.
</details>

## MEDIUM MCQs

### Q6. How do you add a custom error message to `ModelState` manually?
A. `ModelState.Add("Error")`
B. `ModelState.AddModelError("Key", "Message")`
C. `Request.Errors.Add()`
D. `ViewBag.Error = ...`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
This is necessary for business logic validation (e.g., "Email already exists in database") which attributes can't check easily.
</details>

### Q7. What is `asp-validation-summary="ModelOnly"`?
A. Displays all errors.
B. Displays only **Model-Level** errors (where key is empty string), suppressing property-level errors (which are usually shown next to individual fields).
C. Displays only warnings.
D. Displays nothing.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Common for displaying "Invalid Login Attempt" or similar global errors at the top of the form, without repeating every field error.
</details>

### Q8. To enable Client-Side Validation without a full page postback, what scripts are needed?
A. Only `jquery.js`.
B. `jquery.validate.js` and `jquery.validate.unobtrusive.js`.
C. `angular.js`.
D. `bootstrap.js`.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
The "unobtrusive" library reads the `data-val-*` attributes generated by Tag Helpers and configures the jQuery Validate plugin automatically.
</details>

### Q9. What does the `[Range(10, 20)]` attribute validate?

C. Date range.
D. Array size.

<details>
<summary><strong>Answer</strong></summary>

Works for `int`, `double`, `decimal`.

### Q10. Can you use `{0}` in the `ErrorMessage` string?

## HARD MCQs

### Q11. If `ModelState.IsValid` is false in a POST method, why must you reload data (e.g. Dropdowns) before returning `View(model)`?
A. Because View state is lost.
B. HTTP is stateless. The inputs posted by the user are available in the model, but auxiliary data passed via `ViewBag` or `ViewData` (like dropdown options) during the GET request is NOT preserved in the POST request.
C. It is not required.
D. Security.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
A common bug. If you don't repopulate the `SelectListItem` source, the View will crash or show an empty dropdown when re-rendering to show errors.
</details>

### Q12. What takes precedence: `[DisplayName]` or property name?
A. Property name.
B. `[DisplayName]`.
C. Random.
D. Neither.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Attributes exist to override default convention.
</details>

### Q13. `[DataType(DataType.Date)]` vs `[DisplayFormat(DataFormatString="{0:d}")]`?
A. `DataType` only provides a specific input type (UI hint for `<input type="date">`). `DisplayFormat` controls how the value is formatted as a string (e.g. in Display mode).
B. They are identical.
C. DataType handles validation.
D. DisplayFormat is for editing.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** A
**Explanation:**
`DataType` attributes often don't provide server-side validation logic (e.g. date parsing), they hint the UI.
</details>

### Q14. Can you implement `IValidatableObject` on the Model?
A. No.
B. Yes. It allows the model to implement a `Validate` method for complex cross-property validation logic that runs after attribute validation passes.
C. Only for storage.
D. Only in WPF.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
A powerful way to encapsulate "Start Date must be before End Date" logic within the model itself.
</details>

### Q15. Is client-side validation enough for security?
A. Yes, it saves server resources.
B. No. It can be easily bypassed (e.g. using Postman or disabling JS). **Server-side validation is mandatory** for security and data integrity.
C. Yes, if using modern browsers.
D. Yes, if using HTTPS.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Client-side is for User Experience (UX). Server-side is for Security.
</details>
