## EASY MCQs

### Q1. Which namespace contains the Data Annotation attributes?
A. `System.Web`
B. `System.Data`
C. `System.ComponentModel.DataAnnotations`
D. `System.Validation`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** C
**Explanation:**
This namespace holds core attributes like `[Required]`, `[StringLength]`, `[Key]`, etc.
</details>

### Q2. Which attribute ensures a user must provide a value for a property?
A. `[NotNull]`
B. `[Mandatory]`
C. `[Required]`
D. `[Important]`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** C
**Explanation:**
`[Required]` checks that the field is not null or empty string.
</details>

### Q3. Which attribute validates that a string does not exceed a certain length?
A. `[Max]`
B. `[StringLength]` (or `[MaxLength]`)
C. `[Size]`
D. `[Limit]`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
`[StringLength(50)]` ensures the input is at most 50 characters.
</details>

### Q4. To confirm that "Password" and "ConfirmPassword" fields match, which attribute is used?
A. `[Same]`
B. `[Match]`
C. `[Compare("OtherProperty")]`
D. `[Equal]`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** C
**Explanation:**
`[Compare("Password")]` on the `ConfirmPassword` property ensures both fields have identical values.
</details>

### Q5. Which attribute changes the label text displayed for a property in a View?
A. `[Label]`
B. `[Display(Name="...")]` or `[DisplayName("...")]`
C. `[Text]`
D. `[Caption]`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Instead of showing property name "FirstName", `[Display(Name="First Name")]` renders "First Name" in `<label>` helpers.
</details>

## MEDIUM MCQs

### Q6. Which attribute is used to validate input against a Regular Expression pattern (e.g. email or phone format)?
A. `[Regex]`
B. `[RegularExpression("pattern")]`
C. `[Pattern]`
D. `[Format]`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Allows powerful custom validation logic, e.g., `[RegularExpression(@"\d{10}")]` for a 10-digit number.
</details>

### Q7. What does `[DataType(DataType.Password)]` do?
A. Encrypts the password in the database.
B. Provides a hint to the UI helper to render an `<input type="password">` tag causing characters to be masked. It does NOT perform validation itself.
C. Validates password complexity.
D. Hides it from the server.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
`DataType` attributes are primarily for UI hints (display templates) rather than validation logic.
</details>

### Q8. How do you specify a numeric range for a property (e.g., Age 18-100)?
A. `[Range(18, 100)]`
B. `[Between(18, 100)]`
C. `[Limit(18, 100)]`
D. `[MinMax(18, 100)]`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** A
**Explanation:**
`[Range]` validates that numeric inputs fall within min and max boundaries.
</details>

### Q9. What happens if you define `[Required(ErrorMessage = "Fill this!")]`?
A. It throws an exception with that message.
B. If validation fails, `ModelState` contains an error with the text "Fill this!", which is then displayed to the user via input validation helpers.
C. It logs "Fill this!".
D. It puts "Fill this!" in the textbox.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Customizing error messages provides a better user experience than default messages.
</details>

### Q10. What is `[MetadataType]` used for?
A. To add attributes to partial classes generated by tools (like Entity Framework DB First models) where you cannot edit the auto-generated class file directly.
B. To store metadata in DB.
C. To add comments.
D. To change file type.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** A
**Explanation:**
You create a "Buddy Class" containing the attributes and link it to the partial model class using `[MetadataType]`.
</details>

## HARD MCQs

### Q11. Which helper creates a dropdown list for an Enum property in Razor?
A. `Html.DropDownList`
B. `Html.GetEnumSelectList<TEnum>()` used inside `select asp-items`.
C. `Html.EnumList`
D. `Html.EditorFor`

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
`asp-items="Html.GetEnumSelectList<MyEnum>()"` automatically generates `<option>` tags for each enum value.
</details>

### Q12. Does `[DataType(DataType.Date)]` force date validation?
A. Yes, strictly.
B. No, it mainly serves as a UI hint to render `<input type="date">`. Actual format validation depends on the binder and potential `[RegularExpression]`. The browser may enforce UI constraints.
C. Yes, always MM/DD/YYYY.
D. It converts to string.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
DataType attributes are looser than validation attributes. They guide the HTML generator.
</details>

### Q13. `[DisplayFormat(DataFormatString = "{0:C}")]` is used for:
A. Dates.
B. Currency formatting (e.g., $1,234.00).
C. Centimeters.
D. Celsius.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Standard numeric format string "C" stands for Currency (locale aware).
</details>

### Q14. Can you create a **Custom Validation Attribute**?
A. No.
B. Yes, by inheriting from `ValidationAttribute` and overriding `IsValid`.
C. Yes, by using Javascript.
D. Only in paid versions.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
If built-in rules aren't enough (e.g., `[MustBeOddNumber]`), you can write your own attribute logic.
</details>

### Q15. `[HiddenInput(DisplayValue = false)]` typically renders:
A. Nothing.
B. An `<input type="hidden">`. The property value is sent in the form POST but not shown to the user.
C. A password field.
D. Encrypted text.

<details>
<summary><strong>Answer</strong></summary>
**Correct Answer:** B
**Explanation:**
Useful for IDs (Primary Keys) that need to round-trip to the server but shouldn't be edited by the user.
</details>
