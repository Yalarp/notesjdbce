## MEDIUM MCQs

### Q5. What is the correct sequence of the .NET program execution flow?

A. Source Code -> JIT Compiler -> CLR -> Native Code  
B. Source Code -> C# Compiler -> Native Code -> CLR  
C. Source Code -> C# Compiler -> MSIL -> CLR (JIT) -> Native Code  
D. Source Code -> MSIL -> C# Compiler -> Native Code  

<details>
<summary><strong>Answer</strong></summary>

**Correct Answer:** C

**Explanation:**  
The C# source code is compiled by `csc.exe` into MSIL. Then, the CLR loads the assembly, and the JIT compiler converts MSIL to Native Code at runtime.
</details>

### Q6. Which of the following statements is TRUE regarding the JIT Compiler?

A. It compiles the entire program to native code before execution starts.  
B. It converts MSIL to native code on demand, compiling only called functions.  
C. It converts native code back to MSIL for debugging.  
D. It is not responsible for caching compiled code.  

<details>
<summary><strong>Answer</strong></summary>

**Correct Answer:** B

**Explanation:**  
JIT (Just-In-Time) compilation works on demand. It converts MSIL to native code only for the functions that are called, and caches them for reuse.
</details>

### Q7. What distinguishes Managed Code from Unmanaged Code?

A. Managed Code runs directly on the OS, while Unmanaged Code runs under CLR.  
B. Managed Code is always written in C++, while Unmanaged Code is in C#.  
C. Managed Code executes under the control of CLR, utilizing garbage collection and type safety.  
D. Managed Code does not support cross-language interoperability.  

<details>
<summary><strong>Answer</strong></summary>

**Correct Answer:** C

**Explanation:**  
Managed Code is executed under the control of the CLR and benefits from services like modern memory management (garbage collection) and better security.
</details>

### Q8. In the .NET memory model, where are Reference Types (like objects and strings) stored?

A. Stack  
B. Heap  
C. CPU Registers  
D. GAC  

<details>
<summary><strong>Answer</strong></summary>

**Correct Answer:** B

**Explanation:**  
In the .NET memory model, Reference types (objects, strings, arrays) are stored in the Heap, while Value types (int, float) are stored on the Stack.
</details>

### Q12. Regarding the Stack and Heap memory model, which of the following is correct?

A. Stack memory is garbage collected, while Heap memory is cleaned up when the method returns.  
B. Value types are always allocated on the Heap.  
C. Stack allocation is faster and has a LIFO structure, while Heap allocation is dynamic and garbage collected.  
D. Local variables of Reference types are stored entirely on the Stack.  

<details>
<summary><strong>Answer</strong></summary>

**Correct Answer:** C

**Explanation:**  
The Stack uses a LIFO (Last-In-First-Out) structure and allows for fast allocation of value types. The Heap is used for dynamic allocation of reference types and is managed by the Garbage Collector.
</details>
